# Contributing Guidelines

This repository follows a consistent structure to keep all solutions clean, readable, and easy to navigate.

## File Naming

Use the following format:

platform_problem_name.cpp

Examples:

* cses_message_route.cpp
* lc_200_number_of_islands.cpp
* cf_510B_fox_and_two_dots.cpp

## Coding Standards

* Language: C++17
* Write clean and readable code.
* Use meaningful variable names whenever practical.
* Keep formatting consistent.
* Analyze time and space complexity before implementation.

## Commit Messages

Examples:

* Add BFS template
* Solve CSES Message Route using BFS
* Add graph traversal notes
* Implement Dijkstra template

Avoid vague messages such as:

* update
* fix
* commit
* changes
